# Agent Rules

- 사용자가 명시하여 지시하지 않으면 브라우저 서브에이전트 구동 및 스크래치 패드(임시 진행 메모장) 생성을 통한 검증 프로세스를 생략합니다.
- 사용자가 명시적으로 게임 앱 개발 또는 수정을 지시하기 전까지는 어떠한 게임 앱 관련 빌드(`scripts/build_game.py`), 가스 파일 복사, 또는 코드 수정을 수행하지 않습니다. 스토리 수정 프로세스 중에는 오직 시나리오 대본 마크다운과 스토리북 뷰어 빌드(`scripts/build_storybook.py`)만 대상으로 삼아 작업합니다.
- 스토리 마크다운(.md) 파일을 신규 작성하거나 수정한 경우, 작업을 마치기 전에 반드시 `scripts/build_storybook.py`를 실행하여 구문 오류나 파싱 문제가 없는지 빌드 검증을 수행합니다.
- 스토리 내 수학 공식이나 기호(예: 분수, 음수 기호 등)를 수정할 때는 프로젝트에서 기존에 사용하던 표기법(LaTeX 또는 특정 포맷)을 임의로 변경하지 않고 일관된 형식을 유지합니다.
- 이미지 수정할 경우 폐기된 이미지는 레거시 폴더로 이동시킵니다.
- 이미지 생성 및 최적화 규칙:
  - 원본 이미지를 생성한 후, 해당 이미지를 경량화(용량 압축)한 이미지를 별도로 생성합니다.
  - 웹앱 빌드, 스토리북 컴파일 및 깃허브(GitHub) 푸시 시에는 반드시 이 경량화된 이미지를 사용하도록 합니다.
- 학년별 구글 앱스 스크립트(GAS) 배포 범위 규칙:
  - 특정 학년(예: 중1)의 웹앱만 수정되거나 신규 생성되었을 경우에는 해당 학년 프로젝트(`gas/m1_project`)에 대해서만 `clasp push`를 수행합니다. (중2의 경우 `gas/m2_project`, 중3의 경우 `gas/m3_project`만 수행)
  - 불필요하게 변경사항이 없는 다른 학년의 GAS 원격지에 push를 실행하지 않도록 제한하여, 배포 속도를 최적화하고 예기치 못한 덮어쓰기 위험을 방지합니다.
  - 단, `templates/game.html` 공통 템플릿 수정 등 전체 학년이 빌드 대상이 되는 글로벌 변경의 경우에만 예외적으로 전 학년 프로젝트에 대해 순차적으로 `clasp push`를 수행합니다.
